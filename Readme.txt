FBPS ExtractProcessor release 1.9

Умеет:
1) Конвертировать XML реквестов из ПК ПВД в JSON для отправки в Фасад, используя API Трансформера ФБПС
	- Для конвертации требуется положить XML реквесты в папку Requests в корне проекта, приложение поддерживает поиск по подпапкам. Все находящиеся в Requests xml файлы будут отправлены в трансформер 
	и полученный результат записан в $filename.json, где $filename - исходное имя xml файла
2) Добавлять JSON'ы в S3 в заданную папку
	- Для отправки требуется положить JSON'ы реквестов в папку Requests в корне проекта, приложение поддерживает поиск по подпапкам. Все находящиеся в Requests json файлы будут отправлены в S3 в 
	папку указанную в переменной S3FolderName соответствующей выписки в конфиге config.js приложения.
3) Отправлять запрос в RabbitMQ на заданную выписку, используя предварительно загруженный запрос из заданной папки в Minio
	- Для отправки запросов на выписки приложение собирает названия всех json файлов, находящихся в Requests, записывает на каждый UUID выписки data cache и отсылает сообщения в Rabbit, указывая в 
	качестве пути к реквесту в S3 папку из переменной S3FolderName соответствующей выписки и имя каждого json файла найденного в Requests. Соответственно при запуске этой операции в Requests должны 
	лежать только те json, выписки по которым вам НУЖНЫ! UUID'ы выписок будут выведены в терминал и сохранены в ./requests.txt.
	После запроса выписок каждые 10 секунд производится опрос статуса готовности xml выписки и как только статус будет получен - он вернется в консоль. Таймаут опроса 10 минут
4) Выкачивать сформированные выписки из S3
	- Выкачаны могут быть только выписки, которые были получены последним запросом №3 через ExtractProcessor.
	- Скачанные выписки попадают в папку ./Extracts/$extractName
	- Названия папок выписки соответствует названиям json запросов по которым они были отправлены
5) Валидировать синтаксис json выписок
	- По вызову функции будут валидированы все .json файлы лежащие в папке заданной параметром 'exDir'(по умолчанию в ./Extracts)
	- По окончанию работы функция выведет отчет в консоль
6) Валидировать xml выписок по их xsd схемам
	- По вызову функции из всех .xml файлов в папке заданной параметром 'exDir'(по умолчанию в ./Extracts) будут вычитаны их xsd схемы и далее каждый xml будет валидирован относительно этой схемы
	- По окончанию работы функции в консоль будет выведен отчет
	- ВНИМАНИЕ xsd схемы хранятся в папке ./xsd. Их нельзя перемещать. Некоторые схемы переименованы для соответствия своим названиям из xml выписок. При обновлении схем ОБЯЗАТЕЛЬНО соблюсти 
	прежнее именование схем
	- Перед использованием проверьте что задана правильная версия xsd-схем в config.js
7) Проверить xml выписок на пустые теги
	- По вызову функции она пройдет по всем .xml файлам в папке заданной параметром 'exDir'(по умолчанию в ./Extracts) и произведет поиск строк следующих паттернов: '<тег><тег/>' и '</тег>'
	- Результат поиска будет выведен в консоль
	- ВНИМАНИЕ нахождение тегов не является ошибкой в xml и требует ручной проверки найденных тегов на обязательность по xsd схеме

Перед использованием задайте в config.js в разделе //User Config:
	- Номер получаемой выписки в формате Ex$N 
	- Тестовый стенд. Доступные стенды: HF, HFTest
	- Источник выписки. Доступные источники: nsud, fbhd
 	
После каждого изменения config.js ОБЯЗАТЕЛЬНО нужно СОХРАНИТЬ изменения, иначе они не будут применены при запуске программы.

При необходимости работать с другими стендами => нужно создать новый объект с соответствующими ключами в блоке Конфиги тестовых стендов, по аналогии с имеющимися.

Для запуска приложения необходимо:
	- Иметь установленный bash terminal(в IDE или отдельно, неважно)
	- Установить node.js
	- В bash терминале ввести "node app" из корневой папки проекта
	- Далее следовать текстовым инструкциям в терминале
	- После изменения конфига не забудьте перезапустить приложение
